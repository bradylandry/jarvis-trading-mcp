## High

### `JARVIS_TRADING_MAX_CALLS_PER_MIN` env var causes unhandled crash on startup
- **File**: `friend_mcp/server.py:68`
- **Issue**: `int(os.environ.get("JARVIS_TRADING_MAX_CALLS_PER_MIN", "30"))` is evaluated at module import time. If a user sets this env var to a non-integer value (e.g. `""`, `"auto"`, or a typo), Python raises `ValueError` before `main()` is even reached. The MCP process exits silently with no JSON-RPC error — Claude just sees a dead server and reports it as unavailable.
- **Fix**:
  ```python
  try:
      RATE_LIMIT_MAX_CALLS = int(os.environ.get("JARVIS_TRADING_MAX_CALLS_PER_MIN", "30"))
  except ValueError:
      RATE_LIMIT_MAX_CALLS = 30
  ```

### `get_fit_score` silently drops the `opt_type` parameter
- **File**: `friend_mcp/server.py:175-183`
- **Issue**: The function signature is `get_fit_score(ticker: str, opt_type: str = "call")`, but the API call only passes `{"tickers": ticker.upper()}` to `/api/screener`. The `opt_type` argument is accepted, shown to Claude in the tool schema, but never forwarded to the backend. A call like `get_fit_score(ticker="NVDA", opt_type="put")` silently returns call-direction scores instead of put-direction scores, with no indication of the mismatch.
- **Fix**: Pass `opt_type` to the API params dict:
  ```python
  return _api_get("/api/screener", {"tickers": ticker.upper(), "type": opt_type})
  ```
  (Verify the screener endpoint's expected param name against the actual API spec.)

### `serverInfo.version` hardcoded at "0.1.0" while package is at "0.3.0"
- **File**: `friend_mcp/server.py:333`
- **Issue**: The `initialize` response returns `"serverInfo": {"name": "friend_mcp", "version": "0.1.0"}` but `pyproject.toml` declares `version = "0.3.0"`. MCP clients and diagnostics that inspect `serverInfo.version` see a stale value. This also creates confusion when debugging version-specific issues.
- **Fix**: Read the version from the package metadata or a shared constant:
  ```python
  from friend_mcp import __version__
  # ...
  "serverInfo": {"name": "friend_mcp", "version": __version__}
  ```
  And set `__version__ = "0.3.0"` in `friend_mcp/__init__.py` (currently empty).
